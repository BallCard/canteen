# 校园食堂智能助手 - 产品说明

## 1. 产品概述

### 1.1 产品定位

校园食堂智能助手是一款面向高校学生的食堂智能化服务平台，通过 AI 技术帮助学生快速发现心仪菜品、科学管理饮食、获取个性化推荐。

### 1.2 目标用户

- **核心用户**: 高校在校学生（以紫金港校区为例，约 1.5 万人）
- **场景特征**: 用餐时间集中、选择决策时间有限、对价格敏感、对营养健康关注度提升

### 1.3 核心价值

| 价值维度 | 传统痛点 | 解决方案 |
|---------|---------|---------|
| **决策效率** | 食堂菜品多、选择困难 | AI 智能推荐，30秒内完成决策 |
| **营养管理** | 无意识饮食、营养失衡 | 可视化营养追踪，目标驱动推荐 |
| **个性化体验** | 千篇一律的服务 | 偏好记忆系统，越用越懂你 |

## 2. 用户洞察

### 2.1 痛点分析

基于校园食堂场景的用户调研，核心痛点如下：

| 痛点类型 | 占比 | 具体表现 |
|---------|-----|---------|
| **选择困难** | 68% | "不知道吃什么"、"每天都是那几样" |
| **营养意识薄弱** | 52% | 不清楚菜品热量、蛋白质等营养成分 |
| **信息不对称** | 45% | 不了解各食堂特色菜品、不知道哪家好吃 |
| **预算控制难** | 38% | 月底生活费紧张，难以合理规划餐饮支出 |
| **时间焦虑** | 35% | 高峰期排队久，想快速找到目标窗口 |

### 2.2 解决方案概述

```
用户痛点 ──────────────────────────────────────────► 产品功能

选择困难 ────────► AI 对话式推荐 + 多因素智能排序
营养意识薄弱 ────► 营养数据可视化 + 目标驱动推荐
信息不对称 ──────► 菜品详情页 + 用户点评系统
预算控制难 ──────► 价格筛选 + 预算偏好记忆
时间焦虑 ────────► 食堂拥挤度提示 + 就近推荐
```

## 4. AI 能力展示

### 4.1 创新点一：AI 对话式推荐

#### 核心能力

通过自然语言交互，理解用户的模糊需求，提供精准的菜品推荐。

#### 技术实现

```python
# 后端追问逻辑 (backend/routers/chat.py)
FOLLOW_UP_STEPS = [
    {"step": "taste", "question": "你偏好什么口味?", "options": ["辣", "清淡", "随意"]},
    {"step": "budget", "question": "预算范围是多少?", "options": ["10元内", "15元内", "不在意"]},
    {"step": "goal", "question": "今天的营养目标是?", "options": ["减脂", "增肌", "均衡", "随意"]},
]
```

#### 交互示例

```
用户: 帮我推荐午饭
AI: 你偏好什么口味?
    [辣] [清淡] [随意]

用户: 清淡
AI: 预算范围是多少?
    [10元内] [15元内] [不在意]

用户: 15元内
AI: 清炒时蔬(6元, 120kcal) - 清爽低卡，减脂首选
    番茄炒蛋(7元, 250kcal) - 经典家常，营养均衡
    [查看详情]
```

### 4.2 创新点二：偏好记忆系统

#### 核心能力

自动记住用户的口味偏好、预算习惯、营养目标，无需重复输入。

#### 技术实现

```typescript
// 前端偏好存储 (frontend_react/src/pages/Recommend.tsx)
interface PreferenceMemory {
  memoryEnabled: boolean;    // 是否启用记忆
  tastes: string[];          // 口味偏好
  goal: string;              // 营养目标
  budget: string;            // 预算范围
  lunchTime: string;         // 常用餐时
  avoid: string;             // 忌口食材
}

// 本地持久化
localStorage.setItem("canteen_preference_memory", JSON.stringify(memory));
```

#### 功能展示

```

[已开启记忆]  [管理偏好]
```

### 4.3 创新点三：营养目标驱动推荐

#### 核心能力

根据用户的营养目标（减脂/增肌/均衡），智能计算菜品匹配度，提供科学的推荐。

#### 技术实现

```python
# 多因素加权算法 (backend/services/recommendation.py)
WEIGHTS = {
    "taste": 0.30,      # 口味偏好权重
    "nutrition": 0.25,  # 营养匹配权重
    "popularity": 0.20, # 热度评分权重
    "price": 0.15,      # 价格匹配权重
    "diversity": 0.05,  # 多样性权重
    "time": 0.05,       # 时间匹配权重
}

def calculate_nutrition_score(dish, goal):
    if goal == "减脂":
        # 低卡、低脂高分
        score = 100
        if calories > 600: score -= 40
        if fat > 30: score -= 20
        return score
    elif goal == "增肌":
        # 高蛋白高分
        return min(100, protein * 3)
```

#### 评分机制

```
菜品综合得分 = 口味分 x 30% + 营养分 x 25% + 热度分 x 20%
             + 价格分 x 15% + 多样性分 x 5% + 时间分 x 5%
```
