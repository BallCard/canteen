# 产品需求文档（PRD）

## 1. 产品概述

### 1.1 产品定位
校园食堂智能助手，基于 QQ 小程序 + 腾讯混元 AI，解决大学生"不知道吃什么"的决策疲劳，提供菜单查询、智能推荐、营养追踪、UGC 点评功能。

### 1.2 目标用户
**核心用户**：浙大紫金港校区学生（1.5 万人）
- 选择困难症人群："今天吃什么？"
- 健身人群："这道菜多少卡路里？"
- 美食探索者："哪个食堂有新菜？"

**次要用户**：
- 校方后勤部门（收集学生反馈）
- 食堂承包商（了解菜品受欢迎度）

### 1.3 核心价值
- **用户价值**：减少决策时间、科学管理饮食、发现优质菜品
- **校方价值**：收集真实反馈、优化菜品结构、提升满意度
- **腾讯价值**：展示 QQ 生态 + 混元 AI 能力、积累校园场景数据

## 2. 功能需求

### 2.1 核心功能（MVP）

#### 功能 1：今日菜单查询
**用户故事**：作为学生，我想查看今天各食堂的菜单，以便快速了解有哪些选择。

**功能描述**：
- 用户选择食堂 → 显示今日所有菜品
- 展示信息：菜名、价格、照片、营养成分（卡路里、蛋白质、脂肪、碳水）
- 支持搜索：输入菜名关键词快速定位

**验收标准**：
- 覆盖 3 个主要食堂（风味、大食堂、麦香园）
- 每个食堂至少 30 道菜品
- 菜品信息完整度 ≥ 90%（照片、价格、营养数据）
- 页面加载时间 < 2 秒

---

#### 功能 2：AI 智能推荐
**用户故事**：作为学生，我想根据口味和营养目标获得推荐，以便快速做出决策。

**功能描述**：
- 输入：口味偏好（辣/清淡/荤素）、营养目标（减脂/增肌/均衡）
- 输出：推荐 3 道菜 + 理由（"高蛋白低脂，适合增肌"）
- 一键分享到 QQ 群："今天吃麻辣香锅，组队去！"

**推荐算法**：
1. 过滤：排除用户不喜欢的食材
2. 打分：口味匹配度（40%）+ 营养目标匹配度（60%）
3. 多样性：确保推荐的 3 道菜不重复类型（主食/荤菜/素菜）
4. 生成理由：调用腾讯混元生成自然语言解释

**验收标准**：
- 推荐响应时间 < 3 秒
- 推荐理由可读性强（用户理解率 ≥ 80%）
- 分享卡片样式美观（包含菜品图片 + 推荐理由）

---

#### 功能 3：营养追踪打卡
**用户故事**：作为健身人群，我想记录每日饮食并查看营养摄入，以便管理体重。

**功能描述**：
- 拍照上传今日饮食 → AI 识别菜品 → 自动计算营养
- 手动添加：从菜单选择菜品 + 调整份量
- 周报：本周摄入总卡路里、蛋白质、脂肪、碳水，对比目标
- QQ 空间动态：晒打卡成就（"本周减脂成功，消耗 3500 卡 💪"）

**AI 识别流程**：
1. 用户上传照片 → 腾讯云图像识别
2. 返回候选菜品列表（Top 3）
3. 用户确认 → 查询营养数据库 → 计算营养
4. 保存到饮食记录

**验收标准**：
- 菜品识别准确率 ≥ 70%（Top 3 包含正确答案）
- 营养计算误差 ≤ 15%（基于标准份量）
- 周报生成时间 < 1 秒

---

#### 功能 4：UGC 点评与举报
**用户故事**：作为学生，我想分享菜品体验并查看他人评价，以便做出更好的选择。

**功能描述**：
- 上传菜品照片 + 评分（1-5 星）+ 文字评价
- 查看菜品详情页：平均评分、评价列表、照片墙
- 举报功能：食品安全问题 → 直达校方后勤
- 积分激励：上传菜品 +10 分，被采纳 +50 分，月榜前 10 名获得食堂优惠券

**内容审核**：
- 文字审核：腾讯云内容安全（过滤敏感词）
- 图片审核：腾讯云图像审核（过滤违规图片）
- 人工复审：举报内容 24 小时内处理

**验收标准**：
- 上传成功率 ≥ 95%
- 审核响应时间 < 5 秒
- 举报处理时效 < 24 小时

## 3. 用户流程

### 3.1 核心流程图

```
[打开 QQ] → [扫小程序码/搜索"浙大食堂助手"]
    ↓
[首页：今日推荐 + 食堂列表]
    ↓
[选择食堂] → [查看菜单] → [点击菜品查看详情]
    ↓
[AI 推荐] ← 输入口味偏好
    ↓
[分享到 QQ 群] 或 [添加到今日饮食]
    ↓
[营养追踪页] → 查看今日/本周摄入
    ↓
[上传菜品] → 拍照 + 评分 → 获得积分
```

### 3.2 关键页面

**首页**：
- 顶部：今日推荐（3 道菜，轮播）
- 中部：食堂列表（卡片式，显示距离、营业时间）
- 底部：导航栏（首页、推荐、追踪、我的）

**菜单页**：
- 顶部：食堂名称 + 搜索框
- 中部：菜品列表（网格布局，显示照片、菜名、价格）
- 底部：筛选（按价格、营养、评分）

**菜品详情页**：
- 顶部：菜品照片（轮播）
- 中部：价格、营养成分、评分、评价列表
- 底部：添加到今日饮食、分享到 QQ 群

**AI 推荐页**：
- 顶部：输入口味偏好（多选）、营养目标（单选）
- 中部：推荐结果（3 道菜，卡片式，显示推荐理由）
- 底部：重新推荐、分享到 QQ 群

**营养追踪页**：
- 顶部：今日摄入（卡路里、蛋白质、脂肪、碳水）
- 中部：饮食记录（时间轴，显示每餐菜品）
- 底部：拍照识别、手动添加、查看周报

## 4. 非功能需求

### 4.1 性能要求
- 页面加载时间 < 2 秒
- AI 推荐响应时间 < 3 秒
- 图片上传成功率 ≥ 95%
- 并发支持：1000 人同时在线

### 4.2 安全要求
- 用户数据加密存储（AES-256）
- API 接口鉴权（JWT Token）
- 内容审核：文字 + 图片双重审核
- 隐私保护：不收集用户真实姓名、学号

### 4.3 兼容性要求
- QQ 小程序版本 ≥ 8.0
- 支持 iOS 13+ / Android 8+
- 适配屏幕尺寸：375px - 414px

### 5.1 核心指标

**用户增长**：

## 6. 风险与应对

### 6.1 数据源风险
**风险**：食堂菜单更新不及时，爬虫失效
**应对**：
- 冷启动：人工录入 50 道高频菜品
- UGC 众包：学生上传今日菜单
- 校方合作：对接官方系统（长期方案）

### 6.2 用户冷启动风险
**风险**：初期用户少，UGC 内容不足
**应对**：
- 地推：食堂门口扫码送优惠券
- KOL 合作：找校园美食博主推广
- QQ 群推广：在各院系群发布

### 6.3 营养数据准确性风险
**风险**：食堂菜品份量不标准，营养计算误差大
**应对**：
- 标注"估算值，仅供参考"
- 用户可手动调整份量
- 长期：和食堂合作，提供标准化营养标签

### 6.4 校方态度风险
**风险**：学校可能担心负面评价影响食堂
**应对**：
- 强调"建设性反馈"，不是吐槽平台
- 举报功能直达后勤，帮助校方改进
- 展示数据价值：学生喜好分析、菜品优化建议

## 7. 成功指标

### 7.1 短期目标（1 个月）
- 注册用户 ≥ 1000 人（浙大紫金港校区学生总数的 6.7%）
- DAU ≥ 300 人
- AI 推荐使用次数 ≥ 500 次/天
- UGC 上传菜品 ≥ 100 道

### 7.2 中期目标（3 个月）
- 注册用户 ≥ 5000 人（33%）
- DAU ≥ 1500 人
- 菜品库 ≥ 300 道（覆盖所有食堂）
- 校方合作：对接官方菜单系统

### 7.3 长期目标（1 年）
- 扩展到浙大其他校区（玉泉、西溪、华家池、之江）
- 扩展到杭州其他高校（浙工大、杭电、浙财）
- 注册用户 ≥ 5 万人
- 探索商业化：食堂广告、优惠券分发

## 8. 附录

### 8.1 竞品分析

| 产品 | 优势 | 劣势 |
|------|------|------|
| 美团外卖 | 覆盖广、配送快 | 校内禁入、价格高 |
| 饿了么 | 优惠多 | 校内禁入、价格高 |
| 学校官方小程序 | 数据准确 | 功能单一、无推荐 |

**差异化优势**：
- AI 智能推荐（腾讯混元）
- 营养追踪（健身人群刚需）
- UGC 点评（真实反馈）
- QQ 生态集成（分享、空间动态）

### 8.2 技术栈选型理由

**前端：QQ 小程序**
- 原因：目标用户是大学生，QQ 渗透率高
- 优势：无需下载、分享便捷、开发成本低

**后端：Python + FastAPI**
- 原因：轻量、快速、适合 AI 集成
- 优势：异步支持、自动生成 API 文档

**数据库：PostgreSQL**
- 原因：支持全文搜索、JSON 字段
- 优势：开源、稳定、扩展性强

**AI 能力：腾讯混元 + 腾讯云**
- 原因：比赛要求展示腾讯 PCG 能力
- 优势：API 稳定、文档完善、免费额度
