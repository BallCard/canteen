# QQ 小程序前端开发任务

## 项目概述
开发"浙大食堂助手" QQ 小程序前端界面，基于产品需求文档（PRD.md）实现核心功能页面。

## 技术栈
- **平台**：QQ 小程序（原生开发）
- **UI 框架**：WeUI（腾讯官方组件库）
- **开发工具**：QQ 小程序开发者工具
- **语言**：JavaScript / WXML / WXSS

## 核心页面需求

### 1. 首页（index）
**布局结构**：
```
┌─────────────────────────┐
│  顶部导航栏：浙大食堂助手  │
├─────────────────────────┤
│  今日推荐（轮播卡片）      │
│  ┌───────┐ ┌───────┐    │
│  │ 菜品1  │ │ 菜品2  │    │
│  └───────┘ └───────┘    │
├─────────────────────────┤
│  食堂列表                │
│  ┌─────────────────┐    │
│  │ 风味食堂         │    │
│  │ 距离 500m · 营业中│    │
│  └─────────────────┘    │
│  ┌─────────────────┐    │
│  │ 大食堂           │    │
│  └─────────────────┘    │
└─────────────────────────┘
│  底部导航栏              │
│  首页 推荐 追踪 我的     │
└─────────────────────────┘
```

**功能要点**：
- 今日推荐：3 道菜轮播，显示菜品图片、名称、价格、推荐理由
- 食堂列表：卡片式布局，显示食堂名称、距离、营业状态
- 点击食堂卡片 → 跳转到菜单页
- 底部导航栏：4 个 tab（首页、推荐、追踪、我的）

**API 接口**：
- `GET /api/recommendations/today` - 获取今日推荐
- `GET /api/canteens` - 获取食堂列表

### 2. 菜单页（menu）
**布局结构**：
```
┌─────────────────────────┐
│  ← 风味食堂        🔍    │
├─────────────────────────┤
│  筛选：价格 营养 评分    │
├─────────────────────────┤
│  菜品网格（2列）         │
│  ┌────┐  ┌────┐         │
│  │图片│  │图片│         │
│  │菜名│  │菜名│         │
│  │¥12 │  │¥15 │         │
│  └────┘  └────┘         │
│  ┌────┐  ┌────┐         │
│  │图片│  │图片│         │
│  └────┘  └────┘         │
└─────────────────────────┘
```

**功能要点**：
- 顶部搜索框：输入菜名关键词搜索
- 筛选器：按价格（低到高/高到低）、营养（高蛋白/低脂）、评分排序
- 菜品网格：2 列布局，显示菜品图片、名称、价格、评分
- 点击菜品 → 跳转到菜品详情页

**API 接口**：
- `GET /api/dishes?canteen_id={id}` - 获取食堂菜单
- `GET /api/dishes/search?keyword={keyword}` - 搜索菜品

**API 接口**：
- `POST /api/recommendations` - 请求 AI 推荐
  ```json
  {
    "taste_preference": ["spicy", "meat"],
    "nutrition_goal": "lose_weight"
  }
  ```

## 交付要求

1. **代码结构清晰**：按页面组织目录，每个页面独立文件夹
2. **组件复用**：抽取公共组件（菜品卡片、底部导航栏等）
3. **响应式布局**：适配不同屏幕尺寸（375px - 414px）
4. **加载状态**：显示 loading 动画，避免白屏
5. **错误处理**：网络请求失败时显示友好提示
6. **性能优化**：图片懒加载、列表虚拟滚动

---

## 注意事项

1. **API 地址**：开发阶段使用 mock 数据，后端完成后替换为真实 API
2. **图片资源**：使用占位图（https://via.placeholder.com/150），后期替换为真实图片
3. **用户授权**：首次使用时请求用户授权（获取昵称、头像）
4. **分享功能**：测试时需要在真机上测试，开发者工具可能不支持
5. **性能监控**：使用 QQ 小程序性能监控工具，确保页面加载时间 < 2 秒
