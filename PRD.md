# 浙大食堂助手 - 产品需求文档（PRD）

## 1. 产品概述

### 1.1 产品定位
基于 QQ 小程序的校园食堂智能助手，结合腾讯混元 AI 提供菜品推荐、营养追踪、UGC 点评功能。

### 1.2 目标用户
- **核心用户**：浙江大学紫金港校区学生（1.5 万人）
- **次要用户**：校方后勤部门、食堂承包商

### 1.3 核心价值
- 解决"今天吃什么"的决策疲劳
- 提供营养追踪工具（减脂/增肌人群）
- 建立学生 → 校方的反馈闭环
- 展示腾讯 PCG 能力（QQ 生态 + 混元 AI）

---

## 2. 用户场景

### 场景 1：选择困难症
**用户**：大二学生小李
**痛点**：每天中午站在食堂门口纠结 10 分钟
**解决方案**：打开小程序 → AI 推荐 3 道菜 → 直接去对应窗口

### 场景 2：健身人群
**用户**：健身爱好者小王
**痛点**：不知道食堂哪道菜高蛋白低脂
**解决方案**：查看菜品营养成分 → 选择符合增肌目标的菜品 → 记录今日饮食

### 场景 3：美食探索者
**用户**：吃货小张
**痛点**：不知道哪个食堂出新菜了
**解决方案**：首页显示"今日新菜" → 查看其他学生的点评和照片

## 3. 功能列表

### 3.1 核心功能（MVP）

#### 功能 1：今日菜单查询
**优先级**：P0
**描述**：
- 用户选择食堂 → 显示今日所有菜品
- 展示：菜名、价格、照片、营养成分（卡路里、蛋白质、脂肪、碳水）
- 支持搜索：输入"麻辣"查找所有麻辣类菜品

**数据来源**：
- 阶段 1：爬虫 + OCR（浙大食堂公众号菜单图片）
- 阶段 2：UGC 众包（学生上传）
- 阶段 3：校方合作（对接食堂管理系统）

**验收标准**：
- 覆盖 3 个主要食堂（风味、大食堂、麦香园）
- 至少 50 道高频菜品
- 菜品信息完整度 > 80%（有照片、价格、营养数据）

---

#### 功能 2：AI 智能推荐
**优先级**：P0
**描述**：
- 用户输入口味偏好（辣/清淡/荤素）+ 营养目标（减脂/增肌/均衡）
- AI 推荐 3 道菜 + 推荐理由
- 支持分享到 QQ 群："今天吃麻辣香锅，组队去！"

**技术方案**：
- 调用腾讯混元大模型 API
- 输入：用户画像 + 今日菜单 + 营养数据
- 输出：推荐菜品 + 理由（"高蛋白低脂，适合增肌"）

**验收标准**：
- 推荐准确率 > 70%（用户点击推荐菜品的比例）
- 响应时间 < 2 秒
- 推荐理由可读性强（不是机器生成的废话）

---

#### 功能 3：营养追踪打卡
**优先级**：P1
**描述**：
- 用户拍照上传今日饮食 → AI 识别菜品 → 自动计算营养
- 周报：本周摄入 12000 卡，达成减脂目标
- 支持分享到 QQ 空间："本周减脂成功，消耗 3500 卡 💪"

**技术方案**：
- 调用腾讯云图像识别 API（识别菜品）
- 查询营养数据库（基于《中国食物成分表》）
- 估算份量：默认 1 份 = 200g，用户可调整

**验收标准**：
- 菜品识别准确率 > 60%
- 营养计算误差 < 20%
- 支持手动修改识别结果

---

#### 功能 4：UGC 点评与举报
**优先级**：P1
**描述**：
- 学生上传菜品照片 + 评分（1-5 星）+ 文字评价
- 举报功能：食品安全问题 → 直达校方后勤
- 积分激励：上传菜品 +10 分，月榜前 10 名获得食堂优惠券

**数据审核**：
- 图片审核：调用腾讯云内容安全 API（过滤色情、暴力、政治敏感）
- 文字审核：敏感词过滤 + 人工抽查

**验收标准**：
- 每个菜品至少有 3 条点评
- 举报响应时间 < 24 小时（校方承诺）
- 积分系统稳定运行，无刷分漏洞

---

### 3.2 扩展功能（后续迭代）

#### 功能 5：食堂人流预测
**描述**：基于历史数据预测各食堂的排队时长
**技术方案**：时间序列预测模型（LSTM）

#### 功能 6：社交功能
**描述**：找饭搭子（匹配作息相同的同学一起吃饭）
**技术方案**：基于用户画像的推荐算法

#### 功能 7：食堂优惠券
**描述**：和食堂合作，发放电子优惠券
**商业模式**：食堂按核销量付费

## 4. 用户流程

### 4.1 首次使用流程
```
[打开 QQ] → [搜索"浙大食堂助手"或扫码]
    ↓
[授权登录（QQ 一键登录）]
    ↓
[填写口味偏好问卷]（3 道选择题：辣度、荤素、营养目标）
    ↓
[进入首页]
```

### 4.2 日常使用流程
```
[打开小程序]
    ↓
[首页：今日推荐 + 食堂列表]
    ↓
[选择食堂] → [查看菜单] → [点击菜品查看详情]
    ↓
[AI 推荐] ← 输入"我想吃辣的"
    ↓
[分享到 QQ 群] 或 [添加到今日饮食]
    ↓
[营养追踪页] → 查看今日/本周摄入
    ↓
[上传菜品] → 拍照 + 评分 → 获得积分
```

## 5. 数据埋点方案

### 5.1 核心指标

**用户增长**：
- DAU（日活跃用户数）
- 新增用户数
- 留存率（次日留存、7 日留存）

**功能使用**：
- 菜单查询次数
- AI 推荐使用次数
- 营养追踪打卡次数
- UGC 上传次数

**用户行为**：
- 平均停留时长
- 分享到 QQ 群/空间的次数
- 推荐菜品点击率

### 5.2 埋点事件

| 事件名称 | 触发时机 | 参数 |
|---------|---------|------|
| `page_view` | 进入任意页面 | `page_name` |
| `dish_view` | 查看菜品详情 | `dish_id`, `canteen_id` |
| `recommend_click` | 点击 AI 推荐 | `user_preference`, `recommended_dishes` |
| `meal_log` | 记录今日饮食 | `dish_id`, `calories` |
| `review_submit` | 提交点评 | `dish_id`, `rating`, `has_image` |
| `share_qq` | 分享到 QQ | `share_type` (群/空间) |

## 6. 结合腾讯 PCG 业务

### 6.1 QQ 生态集成

**QQ 群分享**：
- 推荐菜品 → 一键分享到班级群/宿舍群
- 卡片样式：菜品图片 + "今天吃这个！" + 小程序跳转

**QQ 空间动态**：
- 营养打卡 → 自动生成图文动态
- 模板："本周减脂成功，消耗 3500 卡 💪"

**QQ 小世界**（短视频）：
- 食堂探店视频、菜品测评
- 话题标签：#浙大食堂 #今天吃什么

### 6.2 腾讯混元 AI 展示

**智能推荐**：
- 基于用户画像 + 实时菜单
- 对话式交互："我想吃辣的，但要低卡" → AI 推荐 3 道菜

**营养分析**：
- 拍照识别 → 生成营养报告
- "这道菜含有 500 卡，蛋白质 30g，适合增肌"

## 7. 风险与应对

### 风险 1：数据源不稳定
**问题**：食堂菜单更新不及时，爬虫失效
**应对**：
- 冷启动阶段：人工录入 50 道高频菜品
- UGC 众包：学生上传今日菜单
- 校方合作：对接官方系统（长期方案）

### 风险 2：用户冷启动
**问题**：初期用户少，UGC 内容不足
**应对**：
- 地推：食堂门口扫码送优惠券
- KOL 合作：找校园美食博主推广
- QQ 群推广：在各院系群发布

### 风险 3：营养数据准确性
**问题**：食堂菜品份量不标准，营养计算误差大
**应对**：
- 标注"估算值，仅供参考"
- 用户可手动调整份量
- 长期：和食堂合作，提供标准化营养标签

### 风险 4：校方态度
**问题**：学校可能担心负面评价影响食堂
**应对**：
- 强调"建设性反馈"，不是吐槽平台
- 举报功能直达后勤，帮助校方改进
- 展示数据价值：学生喜好分析、菜品优化建议

## 9. 竞品分析

### 9.1 现有方案

**浙大钉（官方 APP）**：
- 优势：官方背书，用户基数大
- 劣势：只有食堂位置、营业时间，无菜品信息

**美团/饿了么**：
- 优势：外卖配送
- 劣势：校内禁入或限制，不覆盖食堂档口

**学生自发的小程序/表格**：
- 优势：学生自己做的，接地气
- 劣势：数据更新慢、无人维护、用户少

### 9.2 差异化优势

| 维度 | 浙大钉 | 美团 | 学生小程序 | 我们 |
|------|--------|------|-----------|------|
| 菜品信息 | ✗ | ✗ | △ | ✓ |
| AI 推荐 | ✗ | ✗ | ✗ | ✓ |
| 营养追踪 | ✗ | ✗ | ✗ | ✓ |
| UGC 点评 | ✗ | ✓ | △ | ✓ |
| QQ 生态 | ✗ | ✗ | ✗ | ✓ |
